1.上位机和FPGA都是小端序，以太网UDP规定必须使用大端序，拼包是必须颠倒

2. 数据包总布局

### PA模式 (1024字节总长)

```

+------------------+----------------+---------------------+

|    包头(~52B)    |   数据区(896B)   |   填充/校验(~76B)    |

+------------------+----------------+---------------------+

```
只要一包数据不是 1024 字节，它就直接丢弃
不需要算复杂的动态包长。只需要一个计数器：

- 数 52 个字节（或 64 字节对齐）发包头。
    
- 接着数 896 个字节发 `DAS_Core` 算出来的波束数据。
    
- 凑够 1024 个字节，我们直接把 AXI-Stream 的 `TLAST` 信号拉高（告诉 DMA 这一包结束了，搬去内存）。

3.根据附录里的 `HeaderData` 结构体，我们需要在 FPGA 里建一个类似只读存储器（ROM）或者寄存器组的东西。

- 有些字段是**死数字**：比如 `board_id`，我们直接在 Verilog 里写死为 1。闸门状态 `iGateState`，我们直接填 0。
    
- 有些字段是**活数字**：比如 `frame_id`（帧号），我们需要在 FPGA 里做一个加法器，每发完一包，帧号就 +1，这样上位机就知道有没有丢包。

包头定义
```cpp

#pragma pack(push)

#pragma pack(4)

typedef struct _THeaderData{

    unsigned char  board_id;        // 板卡Id

    unsigned int   frame_id;        // 帧Id

    unsigned short unionId;         // BEAM编号

    unsigned short segmentId;       // 段ID

    unsigned short piontNum;        // 采样点数

    unsigned char  iGateState;      // 闸门I状态

    unsigned short iGateData;       // 闸门I峰值数据

    unsigned int   iGateSite;       // 闸门I位置

    unsigned char  aGateState;      // 闸门A状态

    unsigned short aGateData;       // 闸门A峰值数据

    unsigned int   aGateSite;       // 闸门A位置

    unsigned char  bGateState;      // 闸门B状态

    unsigned short bGateData;       // 闸门B峰值数据

    unsigned int   bGateSite;       // 闸门B位置

    int            encodeX;         // 编码器A计数值

    int            encodeY;         // 编码器B计数值

} HeaderData;

#pragma pack(pop)
```
