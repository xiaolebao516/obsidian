## 2. 设计后加入量化用于FPGA
要将浮点数转化为16位有符号整数 由于无权限查找mathwork资源 使用以下代码进行量化并生成coe文件，就在当前文件夹

```matlab
% 1. 获取刚才导出的浮点系数
h = Num; 

% 2. 手动定点化：乘以 2^15-1 并四舍五入（转换为 16-bit 有符号整数）
h_fixed = round(h * 32767);

% 3. 强行拼装成 Vivado 识别的 .coe 文件格式
fileID = fopen('env_lpf_5mhz.coe', 'w');
fprintf(fileID, 'radix=10;\n');
fprintf(fileID, 'coefdata=\n');
for i = 1:length(h_fixed)
    if i == length(h_fixed)
        fprintf(fileID, '%d;', h_fixed(i)); % 最后一个数字必须以分号结尾
    else
        fprintf(fileID, '%d,\n', h_fixed(i)); % 其余数字以逗号加回车分隔
    end
end
fclose(fileID);
disp('硬件 COE 文件已暴力生成！请在 MATLAB 左侧当前文件夹中查看。');
```

### 一、 频域与核心滤波参数

- **滤波器类型 (Lowpass FIR)：** 采用低通 FIR 结构。**原因：** 提取整流后的低频包络，平滑波形。
    
- **系统采样率 (Fs) = 50MHz：** **原因：** 严格对齐当前 ADC 采样和 FPGA 的全局物理时钟。
    
- **通带截止 (Fpass) = 2MHz：** **原因：** 射频信号经过绝对值整流后，有效的基带包络能量集中在 0~2MHz，必须完整保留。
    
- **阻带起始 (Fstop) = 4MHz：** **原因：** 暴力斩断 5MHz 探头基频的残留，并滤除绝对值非线性操作产生的大量高次谐波噪声。
    
- **滤波器阶数 (Order) = 63 阶：** **原因：** 提供足够陡峭的过渡带（2MHz 到 4MHz 之间），确保高频切得干净。_(注：若后期 DSP48E 资源报警，这是首选的“开刀”对象，可降至 31 阶试水)。_
    

### 二、 硬件实现与接口参数

- **抽取率 (Decimation Rate) = 4：** **原因：** 践行“数字算力换通信带宽”战略。将 50MHz 数据率硬生生压低到 12.5MHz，彻底消除 DMA 和千兆以太网的传输瓶颈。
    
- **输入数据类型 (Input) = Unsigned (无符号)：** **原因：** 极其关键的物理映射。因为上一级是绝对值整流模块，流入 FIR 的数据绝对为正，必须设为无符号数以防止最高位被误判为符号位。
    
- **系数类型 (Coefficient) = Signed 16-bit 定点数：** **原因：** FPGA 无法高效处理浮点数。16 位有符号整数是平衡滤波精度与硬件乘法器消耗的工业界“甜点值”。
    
- **输出位宽 (Output) = 8-bit (截断)：** **原因：** 完美对齐下游 `Data_Packer` 的 8-bit 接口，实现“线对线”直连；同时精准契合最终 B 扫图像 256 级灰度（8-bit）的渲染需求，绝不浪费哪怕一比特的网络带宽。
